# 문제8 불안정한 미션 컴퓨터…

화성 기지 미션 컴퓨터의 시스템 하드웨어 정보 및 실시간 부하 상태를 조회하고, 설정 파일(`setting.txt`)에 따라 원하는 정보만 선택적으로 출력 및 로그로 기록하는 시뮬레이터입니다.

---

## 사용 방법

```bash
python mars_mission_computer.py
```

스크립트를 실행하면 현재 미션 컴퓨터의 시스템 정보와 실시간 부하 상태가 JSON 형태로 터미널에 출력되며, 동시에 `mission.log` 파일에 안전하게 기록(Rotating Logging)됩니다.

**참고:** CPU 실시간 사용량 변화를 테스트하고 싶다면, 코드 하단의 `runComputer.generate_cpu_load()` 주석을 해제하여 인위적인 CPU 부하를 발생시킬 수 있습니다. 부하 발생 모드는 **Ctrl+C**를 입력하여 종료합니다.

---

## 구조 개요

```text
mars_mission_computer.py
├── Config            # 로그 파일명, 설정 파일명 등 상수 관리 클래스
├── SystemProvider    # OS별 시스템 정보 수집을 위한 인터페이스 및 구현체 (전략 패턴)
│   ├── WindowsProvider  # Windows 환경용 정보 수집 (ctypes, wmic 활용)
│   └── DefaultProvider  # 미지원 OS 기본 반환
├── SystemInfo        # OS, CPU, Memory 등 시스템 정보를 반환하는 정적(Static) 클래스
└── MissionComputer   # 정보 필터링, 화면 출력, 파일 로깅을 담당하는 메인 클래스
```

---

## 클래스 설명

### 1. SystemInfo & SystemProvider (시스템 정보 수집)

외부 라이브러리를 사용하지 않고 파이썬 내장 라이브러리(`ctypes`, `subprocess`, `platform`, `os`)만을 활용해 하드웨어 정보를 수집합니다.

- **SystemProvider (전략 패턴 적용)**: 운영체제별로 메모리 크기 및 시스템 부하를 측정하는 방식을 분리하여 유지보수성을 높였습니다. 
  - `WindowsProvider`는 `ctypes`를 통해 윈도우 네티브 API(`GlobalMemoryStatusEx`)를 호출하여 메모리를 측정하며, `wmic` 명령어를 호출해 CPU 실시간 사용량을 가져옵니다.

#### 제공하는 데이터
- `운영체계`, `운영체계 버전`, `CPU의 타입`, `CPU의 코어 수`
- `메모리의 크기` (GB 단위)
- `CPU 실시간 사용량` (%), `메모리 실시간 사용량` (%)

---

### 2. MissionComputer (메인 로직 및 출력 관리)

수집된 데이터를 화면에 보여주고 로그 파일로 남기는 전반적인 흐름을 관리합니다. 단일 책임 원칙(SRP)에 따라 데이터 수집, 화면 출력, 파일 로깅이 세분화되어 있습니다.

#### 주요 메소드

**`get_setting_keys()`**
- `setting.txt` 파일을 읽어들입니다.
- 파일 내에 작성된 JSON 형식의 `true` / `false` 설정값을 파싱하여, 출력이 활성화(`true`)된 키(Key)들의 리스트만 반환합니다.

**`get_mission_computer_info()` / `get_mission_computer_load()`**
- `SystemInfo`를 통해 데이터를 수집하고, `get_setting_keys()`의 필터링 목록과 대조합니다.
- 설정에서 `true`로 지정된 항목(혹은 에러 메시지)만 남긴 딕셔너리를 생성한 뒤 출력 및 로깅 메서드로 넘깁니다.

**`display_data(data_dict)`**
- 파이썬 내장 `json` 모듈을 활용하여 딕셔너리 데이터를 들여쓰기(`indent=4`)가 적용된 깔끔한 JSON 포맷으로 터미널에 출력합니다.

**`record_data(data_dict)`**
- `logging` 모듈과 `RotatingFileHandler`를 활용하여 로그 파일 용량이 무한정 커지는 것을 방지합니다. (최대 5MB, 백업 3개 유지)
- 로그 파일에는 한 줄짜리 JSON 문자열 형태로 안전하게 기록합니다.

## 보너스 과제: 출력 항목 설정 (setting.txt)

프로그램과 동일한 디렉터리에 있는 `setting.txt` 파일을 수정하여, 화면과 로그에 노출할 항목을 동적으로 제어할 수 있습니다. 

```json
info = {
    "운영체계": true,
    "운영체계 버전": true,
    "CPU의 타입": false,
    "CPU의 코어 수": true,
    "메모리의 크기": true
}

load = {
    "CPU 실시간 사용량": true,
    "메모리 실 센시간 사용량": true
}
```

## 예외처리 목록

| 위치 | 처리 내용 |
|------|-----------|
| **메모리/CPU 수집 (`SystemProvider`)** | 윈도우 API 호출이나 서브프로세스(`wmic`) 실행 실패 시 프로그램이 종료되지 않고 '알 수 없음'을 반환하며 내부적으로 에러를 로깅합니다. |
| **전체 정보 수집 단계** | 예상치 못한 오류 발생 시 결과 JSON에 `"error"` 키를 추가하여 사용자가 실패 원인을 확인할 수 있도록 합니다. |
| **`setting.txt` 파싱** | 파일이 없으면 필터링 없이 전체 정보를 반환하며, JSON 파싱 오류 발생 시 해당 내용을 로깅하고 무시합니다. |

## 제약 사항

- **Python 3.x** 환경에서 작동합니다.
- 외부 패키지(`psutil` 등)를 일절 배제하고 오직 **Python 표준 내장 라이브러리**만 사용해 구현되었습니다. (`json`, `time`, `random`, `platform`, `os`, `subprocess`, `ctypes`, `logging`)
- **PEP 8** 스타일 가이드를 준수하여 작성되었습니다.
